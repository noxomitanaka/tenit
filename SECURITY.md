# Security Policy

## サポート対象バージョン

最新の `main` ブランチに対してのみセキュリティ修正を提供します。タグ付きリリースはまだ提供していません（OSS公開準備フェーズ）。

| バージョン | サポート |
|------------|----------|
| `main` (latest) | ✅ |
| その他 | ❌ |

## 脆弱性の報告

セキュリティ脆弱性を発見した場合、**公開 Issue ではなく**以下の方法で報告してください。

### 推奨経路: GitHub Security Advisories

1. このリポジトリの **Security** タブを開く
2. **Report a vulnerability** をクリック
3. 詳細を記入して送信

GitHub Private Vulnerability Reporting を使うことで、修正完了まで非公開で議論できます。

### 代替経路: メール

GitHub アカウントを持たない場合: `noxomitanaka` 宛て GitHub プロフィール経由で連絡してください。

### 含めてほしい情報

- 脆弱性の種類（例: XSS, SQL Injection, Auth bypass）
- 影響を受けるファイル / エンドポイント
- 再現手順 / 概念実証コード
- 想定される影響範囲

## 対応プロセス

1. 受領後 7 日以内に初回応答
2. 脆弱性の検証 / 影響範囲特定
3. パッチ作成 / セキュリティアドバイザリ作成
4. 公開協調日を相談（通常 30〜90 日以内）
5. パッチリリース + アドバイザリ公開 + 報告者クレジット

## スコープ外

以下は本プロジェクトのスコープ外です。

- 自己ホスト環境の設定ミス（HTTPS 未設定、リバースプロキシ不在等）
- 依存ライブラリ自体の脆弱性（上流に報告してください）
- 物理アクセスを前提とした攻撃
- 既知の AGPL ライセンス遵守問題

## 既知の制約

このプロジェクトは個人運営の OSS です。Tier 1 サポートではありませんが、可能な限り迅速に対応します。

商用利用で SLA が必要な場合は Issue で相談してください。
